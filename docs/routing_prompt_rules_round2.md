# Routing Prompt Rules Round2 Draft

以下仅为 `prompt_rules.business_rules` 草案，不直接写库。

## 业务规则草案

1. 同类制程按顺序对位。
多个铜绞对应多个铜绞，多个芯押对应多个芯押，默认按材料/制程出现顺序一一匹配。

2. 外购铜绞不参与下方铜绞制程。
如果某条铜绞是外购来源或已作为外购材料处理，则该条材料不要再匹配给内部铜绞制程费用；但不代表整张表所有铜绞制程都跳过。

3. 对绞金额取值规则。
对绞相关制程优先取以 P 结尾规格的铜绞和芯押作为参与材料；若同一组内无法唯一识别对应的 P 结尾规格，或找不到可靠的铜绞/芯押配对关系，必须 manual_review。

4. 价格缺失不是路由问题。
如果失败原因是铜价为空、单价缺失、PVC/外购价格源缺失，应返回 manual_review 或 reject，不应强行推荐 skill；reason 必须明确说明是价格问题，而不是写成制程不匹配。

5. 多 skill 可能性。
如果一张成本分析表同时需要导体/编织和胶料/后续制程两个阶段，应在 reason 中明确说明“可能需要多阶段计算”；当前 schema 仍只允许选择最主要的 target_skill，不要伪造多个 skill。
